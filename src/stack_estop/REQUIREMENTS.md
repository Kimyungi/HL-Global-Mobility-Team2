# stack_estop — 요구사항

**담당: 박찬미** · 산출물: 긴급 정지 (8/2)

## 역할

돌발 장애물 감지 → 긴급 정지 요구

**시나리오 (대회 규정):** 장애물이 갑자기 튀어나오면 정지 → 장애물이 치워지면 **자동 재출발**.
재출발은 MGM이 처리한다(estop=false가 되면 rate limit으로 재가속) — 이 스택은 "지금 위험한가"만 답하면 된다.

## 계약 (이것만 지키면 나머지는 자유)

- 입력: 2D LiDAR (최우선 반응 경로 — 가능한 한 짧은 주기, 스켈레톤 기본 50ms).
- 출력: `/perception/estop` (`fma_interfaces/EstopRequest`).
  - `estop`은 모든 주행 스테이트 우선권 표의 **최상위 요구**. MGM이 v_ref=0 (rate limit 우회)으로 반영.
  - **정지는 스테이트가 아니다**: estop 중에도 주행 스테이트는 유지되고, 요구 해제 시 자동 복귀.
    따라서 estop 해제 조건(장애물 소멸 확인)을 신중히 — 해제 즉시 차가 다시 출발한다.
- **estop은 레벨 신호다 (래치 금지, 1회 이벤트 금지).** MGM은 매 10ms 현재 값만 본다.
  - 매 스캔마다 "지금 위험한가"를 재평가해서 발행할 것.
  - "감지 순간 1회 발행"으로 구현하면 → 장애물이 아직 앞에 있는데 차가 재출발 (사고).
  - 스택 내부에서 래치를 걸면 → 장애물이 치워져도 재출발 불가 (대회 규정 위반).
- **판정 기준: 전방 주행 통로 안에 물체 + TTC(또는 거리) < 임계 → estop=true. 그게 전부다.**
  - 회피 가능 여부는 **판정하지 않는다** — 그건 stack_avoid(이기돈) 소관. 두 스택이 동시에
    반응해도 안전하다(우선권 표에서 estop이 최상위). **겹침은 안전하고 공백만 위험하다** —
    기준을 넉넉하게 잡을 것. 회피 불가 정적 장애물(길 전체 막음)도 이 기준이 최후 방어선이다.
  - "돌발"을 의미론적으로 구분할 필요 없음 — 갑자기 튀어나오면 TTC가 짧다는 물리량으로 자동 포착된다.
- 금지: 이 스택에서 모터/브리지에 직접 명령 금지. 유일한 출구는 estop 플래그.
- 검증: 감지→토픽 발행 지연 실측 (전체 정지 체인: 감지 → MGM 10ms → TX → dSPACE).
  - 시나리오 검증: 장애물 투입 → 정지 확인 → 제거 → **자동 재출발** 확인까지가 한 세트.

## 공통 규칙 (CLAUDE.md)

- 출력은 `fma_interfaces` 메시지로만. MGM은 이 토픽만 구독한다.
- 경로를 내는 스택은 전부 동일 ref points 포맷 — {x, y, yaw, curvature}, vehicle frame (§5.4).
- 판단 로직(모드 전환·정지 결정·우선권)은 MGM 스테이트 머신에만 존재한다 (§4, §5.1).
- 실행: `ros2 run stack_estop stack_estop_node`
